## 编码器问题

### Q1：编码器读数为0

**现象**：编码器读数一直是0，不变化

**可能原因**：
1. 硬件连接问题
2. 定时器未配置
3. 编码器损坏

**解决方法**：
```c
// 1. 检查硬件连接
//    - A、B相是否接对
//    - 电源是否正常

// 2. 检查定时器配置
printf("TIM2->CR1: 0x%x\n", TIM2->CR1);
printf("TIM2->SMCR: 0x%x\n", TIM2->SMCR);
printf("TIM2->CNT: %d\n", TIM2->CNT);

// 3. 手动转动电机，观察CNT是否变化
```

### Q2：编码器方向反了

**现象**：正转时读数减少，反转时读数增加

**解决方法**：
```c
// 方法1：交换A、B相接线

// 方法2：软件取反
int16_t delta = last_count - current_count;  // 注意顺序反了
```

### Q3：编码器读数抖动大

**现象**：静止时读数也有波动

**可能原因**：
1. 机械振动
2. 电磁干扰
3. 编码器质量问题

**解决方法**：
```c
// 软件滤波
#define FILTER_SIZE 5
float filter_buffer[FILTER_SIZE];
int filter_index = 0;

float filter_rpm(float rpm) {
    filter_buffer[filter_index] = rpm;
    filter_index = (filter_index + 1) % FILTER_SIZE;
    
    float sum = 0;
    for (int i = 0; i < FILTER_SIZE; i++) {
        sum += filter_buffer[i];
    }
    return sum / FILTER_SIZE;
}
```

## PID问题

### Q4：轮速响应太慢

**现象**：实际轮速缓慢接近目标

**原因**：Kp太小

**解决方法**：
```c
// 增大Kp
inner_pid[i].Kp *= 1.5f;
```

### Q5：轮速有静差

**现象**：实际轮速稳定在目标附近，但不等于目标

**原因**：Ki太小或为0

**解决方法**：
```c
// 增大Ki
inner_pid[i].Ki *= 2.0f;
```

### Q6：轮速振荡

**现象**：实际轮速在目标上下波动

**原因**：Kp或Ki太大

**解决方法**：
```c
// 减小Kp和Ki
inner_pid[i].Kp *= 0.7f;
inner_pid[i].Ki *= 0.5f;
```

### Q7：轮速超调大

**现象**：实际轮速先超过目标，再回落

**原因**：Kp太大

**解决方法**：
```c
// 减小Kp
inner_pid[i].Kp *= 0.8f;

// 或增加Kd（速度环通常不用）
inner_pid[i].Kd = 0.01f;
```

## 四轮协同问题

### Q8：四轮转速不一致

**现象**：设置前进运动，但四轮转速有差异

**可能原因**：
1. 电机特性差异
2. PID参数不一致
3. 机械负载不均

**解决方法**：
```c
// 1. 单独调整每个轮子的PID参数
//    观察每个轮子的响应，单独调参

// 2. 添加轮速补偿
float rpm_offset[4] = {0, 2, -1, 1};  // 根据实测调整
target_rpm[i] += rpm_offset[i];
```

### Q9：前进时跑偏

**现象**：机器人前进时向左或向右偏

**可能原因**：
1. 左右轮速不一致
2. 机械安装不对称
3. 地面摩擦不均

**解决方法**：
```c
// 1. 校准左右轮速
//    让机器人前进固定距离，测量偏移量
//    根据偏移量调整左右轮的目标转速

// 2. 添加陀螺仪修正
float heading_error = target_heading - current_heading;
target_omega = heading_error * heading_Kp;
```

### Q10：平移时轨迹弯曲

**现象**：机器人平移时不是直线

**可能原因**：
1. 运动学参数不准确
2. 四轮转速配合不对

**解决方法**：
```c
// 1. 校准运动学参数
//    实测轮距L和轮子半径r

// 2. 校准四轮转速
//    让机器人平移固定距离，观察轨迹
//    调整各轮的目标转速
```

## 外环问题

### Q11：外环不收敛

**现象**：机器人速度无法稳定在目标值

**可能原因**：
1. 外环Kp太大
2. 外环周期太短
3. 内环未调好

**解决方法**：
```c
// 1. 减小外环Kp
outer_pid_Vx.Kp *= 0.5f;

// 2. 增加外环周期
//    外环周期 = 内环周期 × 5~10

// 3. 确保内环已调好
//    内环能稳定跟踪目标轮速
```

### Q12：外环响应太慢

**现象**：机器人速度缓慢接近目标

**原因**：外环Kp太小

**解决方法**：
```c
// 增大外环Kp
outer_pid_Vx.Kp *= 1.5f;
```

## 传感器问题

### Q13：陀螺仪漂移

**现象**：静止时陀螺仪读数不为0

**解决方法**：
```c
// 零偏校准
float gyro_offset = 0;

void gyro_calibrate(void) {
    float sum = 0;
    for (int i = 0; i < 100; i++) {
        sum += mpu6050_get_gyro_z_raw();
        HAL_Delay(10);
    }
    gyro_offset = sum / 100.0f;
}

float get_omega_gyro(void) {
    return mpu6050_get_gyro_z_raw() - gyro_offset;
}
```

### Q14：陀螺仪噪声大

**现象**：陀螺仪读数抖动大

**解决方法**：
```c
// 滤波
float filtered_omega = filter(get_omega_gyro());
```

## 问题诊断流程

```
发现问题
    │
    ▼
观察现象 ──→ 记录数据（波形、日志）
    │
    ▼
分析原因 ──→ 查阅FAQ
    │
    ▼
尝试解决 ──→ 一次只改一个参数
    │
    ▼
验证效果 ──→ 观察是否改善
    │
    ├─→ 改善 → 继续优化
    │
    └─→ 未改善 → 回退，尝试其他方法
```
