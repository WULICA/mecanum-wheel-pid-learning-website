# 编码器原理

> 理解增量式编码器如何测量转速

---

## 什么是编码器？

编码器是一种将机械旋转转换为电信号的传感器。

在电机控制中，我们使用的是**增量式编码器**，它能输出与旋转角度成正比的脉冲信号。

### 编码器的结构

编码器内部有一个刻有光栅的圆盘：

```
        光栅圆盘
           ↓
    ┌─────────────┐
    │  ╲╱╲╱╲╱╲╱  │  ← 透光和不透光交替
    │   ╲╱╲╱╲╱   │
    │    ╲╱╲╱    │
    └─────────────┘
         ↓
    LED + 光敏元件
         ↓
      电脉冲输出
```

当圆盘旋转时，光敏元件会检测到透光和不透光的交替变化，输出方波信号。

---

## 编码器输出什么？

增量式编码器输出**两路正交方波**：A相和B相。

### 信号波形

```
时间轴 →

A相: ─┐ ┌─┐ ┌─┐ ┌─┐ ┌─
      └─┘ └─┘ └─┘ └─┘

B相: ──┐ ┌─┐ ┌─┐ ┌─┐ ┌
        └─┘ └─┘ └─┘ └─┘

相位差: 90°（1/4周期）
```

**关键特征**：
- A相和B相频率相同
- B相滞后A相90°（正转时）
- B相超前A相90°（反转时）

### 为什么需要两相信号？

单相信号只能知道"转了"，不能知道"往哪转"。

两相信号通过相位关系可以判断旋转方向：

```
正转时：
A相上升沿 → B相为低电平
A相下降沿 → B相为高电平

反转时：
A相上升沿 → B相为高电平
A相下降沿 → B相为低电平
```

## 正交解码原理

### 什么是正交解码？

正交解码是通过分析A、B两相信号的相位关系，计算出旋转方向和角度增量的过程。

### 解码方法

**方法一：A相上升沿触发**

```
当A相出现上升沿时：
  读取B相电平
  如果B相为低 → 正转，计数值 +1
  如果B相为高 → 反转，计数值 -1
```

**方法二：四倍频解码（更精确）**

```
A相上升沿、下降沿都触发
B相上升沿、下降沿都触发

每个周期可以检测4次变化：
  A上升沿 + B低 → +1
  A下降沿 + B高 → +1
  B上升沿 + A高 → +1
  B下降沿 + A低 → +1

结果：分辨率提高4倍
```

### 示例

假设编码器每转输出500个脉冲（500 PPR）：

| 解码方式 | 每转计数变化 | 分辨率 |
|---------|-------------|--------|
| 单倍频 | ±500 | 0.72°/count |
| 二倍频 | ±1000 | 0.36°/count |
| 四倍频 | ±2000 | 0.18°/count |

**四倍频解码**是STM32定时器编码器模式的默认方式。

## 编码器参数

### 关键参数

| 参数 | 符号 | 说明 | 典型值 |
|------|------|------|--------|
| 每转脉冲数 | PPR | Pulses Per Revolution | 500, 1000, 2000 |
| 分辨率 | - | 最小可检测角度 | 360°/PPR |
| 最高转速 | RPM_max | 不丢脉冲的最高转速 | 3000~6000 RPM |
| 输出类型 | - | 推挽输出或开漏输出 | 推挽 |

### PPR的选择

PPR越高，分辨率越高，但最高转速会降低。

```
选择原则：
- 低速应用（< 100 RPM）：选高PPR（2000+）
- 中速应用（100~1000 RPM）：选中PPR（500~1000）
- 高速应用（> 1000 RPM）：选低PPR（100~500）
```

对于麦氏轮机器人，通常选择 **500~1000 PPR** 的编码器。

## 编码器与STM32的连接

### 硬件接线

```
编码器          STM32
───────        ──────
A相输出  ───→  TIMx_CH1引脚
B相输出  ───→  TIMx_CH2引脚
VCC      ───→  3.3V或5V（根据编码器）
GND      ───→  GND
```

### 引脚示例（STM32F103）

| 定时器 | CH1引脚 | CH2引脚 |
|--------|---------|---------|
| TIM2 | PA0 | PA1 |
| TIM3 | PA6 | PA7 |
| TIM4 | PB6 | PB7 |

## 常见问题

### 问题1：编码器方向反了怎么办？

**原因**：A、B相接反了

**解决**：交换A、B相的接线，或在软件中取反计数值

### 问题2：高速时丢脉冲怎么办？

**原因**：
- 编码器最高转速不够
- MCU处理速度不够（如果用外部中断方式）

**解决**：
- 选择更高转速等级的编码器
- 使用定时器硬件编码器模式（推荐）

### 问题3：读数抖动大怎么办？

**原因**：
- 机械振动
- 电磁干扰

**解决**：
- 软件滤波（滑动平均）
- 硬件滤波（RC滤波电路）

---

## 本节要点

1. **增量式编码器**输出两路正交方波（A相、B相）
2. **相位关系**决定旋转方向
3. **四倍频解码**可提高4倍分辨率
4. **PPR**决定编码器的分辨率
5. **STM32定时器**可硬件自动完成正交解码
